# Evolastra Observatory shared contract

Status: accepted for the initial implementation on 2026-07-17.

This contract exists to keep independent workstreams compatible. It is deliberately smaller than the product surface: additions may extend it, but must not redefine these meanings.

## Runtime profile

- Backend: Python 3.12, FastAPI, Pydantic v2, SQLAlchemy 2, Alembic.
- Local persistence: SQLite with transactional writes and foreign keys enabled.
- Production persistence target: PostgreSQL through the same SQLAlchemy models.
- Frontend: TypeScript, React, Vite, TanStack Query, a focused external store, and a Canvas 2D renderer isolated from React render frequency.
- Browser live transport: resumable Server-Sent Events using a per-run sequence cursor. Commands use validated HTTP endpoints. This keeps durable delivery unidirectional and command authorization explicit.
- Artifact storage: content-addressed files under a configured local root; clients address opaque artifact IDs, never paths.

## Three layers

1. Operational telemetry records execution facts: traces, spans, logs, and metrics. It uses OpenTelemetry concepts and W3C trace context.
2. The semantic graph records analytical meaning: runs, nodes, agents, datasets, artifacts, claims, evidence, findings, anomalies, approvals, and relationships.
3. The visualization projection contains disposable coordinates, clusters, label priorities, camera state, and coalesced metrics. It can be rebuilt from layer 2 and the event log.

No visualization coordinate or galaxy metaphor may enter a canonical semantic payload.

## Identifier and version rules

- Persisted IDs are UUIDv4 strings with a readable prefix such as `run_`, `node_`, `evt_`, or `art_`.
- `schema_version` is an integer on persisted semantic records.
- CloudEvent `type` values use `galaxy.<area>.<entity>.<action>.v1`.
- Event IDs are globally unique. `sequence` is monotonically allocated per run at ingestion.
- Event correction creates a new event. Stored events are never edited.
- Timestamps are UTC RFC 3339 strings at API boundaries and timezone-aware datetimes internally.

## Durable event envelope

Required CloudEvents-compatible fields:

`specversion`, `id`, `source`, `type`, `subject`, `time`, `datacontenttype`, `dataschema`, `runid`, `sequence`, `traceid`, `spanid`, `correlationid`, `causationid`, `producerversion`, `privacyclass`, and `data`.

The API accepts missing `sequence` only on ingestion, where the server allocates it. Unknown event types are persisted and ignored by projections. Invalid envelopes are quarantined with a redacted reason.

## Initial semantic projection

The vertical slice projects these durable event families:

- run created, started, updated, completed, and failed;
- node created, promoted, started, progressed, completed, and failed;
- agent created, started, status changed, handed off, completed, and failed;
- tool call requested, started, completed, and failed;
- dataset registered and dataset version created;
- artifact created and preview created;
- claim created, validated, and disputed;
- evidence attached;
- finding created and promoted;
- anomaly created and resolved;
- approval requested and recorded;
- metric recorded; and
- snapshot created and heartbeat.

## API namespace

- REST: `/api/v1`
- Health: `/health/live`, `/health/ready`, `/health/database`, `/health/storage`, `/health/projection`
- Stream: `/api/v1/runs/{run_id}/events/stream?after=<sequence>` with `Last-Event-ID` support
- OpenAPI: generated by FastAPI in development
- AsyncAPI: `docs/event-protocol/asyncapi.yaml`

## Trust and privacy defaults

- This initial local profile is single-user and loopback-oriented. Production multi-user authentication and tenant authorization are extension points, not falsely implied.
- Content capture is default-deny for known secret-shaped fields. Redaction happens before persistence and logging.
- CORS is restricted to configured origins. Security headers and trusted hosts are enforced centrally.
- Imported text and artifacts are untrusted data, never instructions. No notebook, HTML, SVG, or generated code is executed.
- Destructive and risky actions require an approval record in the semantic graph.

## Projection determinism

For a fixed ordered event stream and run seed, semantic state and layout are deterministic. Coordinates use run-seeded hashing plus stable radial branch allocation. User pin overrides live in projection storage and do not mutate semantic entities.

## Workstream ownership

- Integration lead: root tooling, backend implementation, frontend implementation, end-to-end integration, and release gate.
- Architecture/standards workstream: `docs/architecture/`, `docs/event-protocol/`, standards fixtures, and schema contract review.
- Visualization/assets workstream: `docs/benchmarks/`, `docs/assets/`, and renderer/asset decisions; it may not replace application code.
- Integrations/security workstream: `integrations/`, `sdk/`, `docs/integration/`, and security review documents; it must consume this envelope rather than invent another.
